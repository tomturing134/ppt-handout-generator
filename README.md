# PPT Handout Generator

从课堂录播视频自动提取 PPT 关键帧 + 逐字稿，融合生成结构化讲义。

## 项目流水线

```
V1：视频 → PPT关键帧提取（Masked SSIM + YOLO 人形检测）
V2：音频 → 逐字稿（faster-whisper）
V3：关键帧 + 逐字稿 → 结构化讲义（Markdown + Word）
  └─ 内嵌图片提取（轮廓检测 + 饱和度分类）
```

## 项目结构

```
├── simple_extractor.py      # PPT内嵌几何图提取器
├── run_extractor.bat        # Windows 双击运行
├── ppt_keyframe_extractor/  # V1 关键帧提取模块
├── audio/                   # V2 音频 + 逐字稿
├── test_output_v10/
│   ├── slides/              # 关键帧原图 (11张)
│   ├── cropped_slides/      # 裁剪后幻灯片
│   └── handout_images/      # 精选几何图输出
├── 讲义_证明.md              # Markdown 讲义
├── 讲义_证明.docx            # Word 讲义
└── pyproject.toml           # 项目配置
```

## 使用方法

1. 将课堂录播视频放在项目目录下
2. 运行 `ppt_keyframe_extractor/` 提取 PPT 关键帧
3. 运行 `audio/` 提取音频并生成逐字稿
4. 双击 `run_extractor.bat` 提取几何图
5. 生成讲义：`node generate_docx.js`

## 技术亮点

- **Masked SSIM**：将老师区域置零后计算 SSIM，避免走动误触发切换
- **饱和度分类**：通过 HSV 饱和度方差区分文字块和几何图
- **逐字稿辅助**：利用 "如图" 关键词降低检测阈值，捕获黑白线段图
- **间距聚类**：自动合并图形区域和附近文字标签（A/B/C/D/O）

## 示例

| 例题 | 提取的几何图 |
|------|-------------|
| OA⊥OC, OB⊥OD → ∠AOB=∠COD | ✅ 角度关系图 |
| BD平分∠ABC, ∠1=∠C → ∠2=∠C | ✅ 角平分线图 |

## 核心技术详解

### V1：关键帧提取 (ppt_keyframe_extractor)

| 技术 | 作用 |
|------|------|
| **Masked SSIM** | 将老师区域置零后对比两帧画面。差异大 → 触发翻页检测 |
| **ONNX YOLO (yolo11n-seg)** | 深度学习实例分割模型(~11.7MB)，精确分割人体轮廓。只 mask 老师身体，保留板书笔迹 |
| **MOG2** | 传统背景减除法，把移动物体当老师 mask 掉。适合遮挡较少的场景，但会连带把板书笔迹也 mask |
| **pHash (感知哈希)** | 将画面压缩成 64 位二进制指纹，比对指纹判断内容变化。适用于白板/滚动式视频（非翻页PPT场景） |
| **GaussianBlur(31x31)** | pHash 预处理：大幅模糊画面，老师写的细线消失，大的图形结构保留 |
| **DCT + 汉明距离** | pHash 的核心：离散余弦变换提取低频特征 → 二进制哈希 → 汉明距离衡量差异位数 |

#### SSIM 阈值参考

| 视频类型 | 推荐阈值 | 说明 |
|----------|---------|------|
| 传统翻页PPT | 0.95 | 只检测大幅切换 |
| 板书/标注多的视频 | 0.75-0.85 | 检测笔迹变化 |

### V2：音频 + 语音转文字

| 技术 | 作用 |
|------|------|
| **imageio-ffmpeg** | 自带 ffmpeg 二进制，从视频中提取音频轨道 → 16kHz 单声道 WAV |
| **faster-whisper (base)** | CTranslate2 加速版语音识别 (~140MB 模型)，输出带时间戳的逐字稿 JSON |

> **注意**：JSON 必须显式指定 `encoding="utf-8"`，否则中文会乱码。

### V3a：PPT 内嵌图片提取 (simple_extractor.py)

这是项目中迭代次数最多的模块（共 8 版），核心流程如下：

#### 核心技术

| 技术 | 作用 |
|------|------|
| **Canny 边缘检测** | 找出画面中所有物体的边缘轮廓（图形边框、文字笔画） |
| **闭运算 (MORPH_CLOSE)** | 膨胀 + 腐蚀，把邻近边缘"粘"在一起形成完整内容块 |
| **HSV 饱和度方差 (sat_std)** | 区分"文字块"(低饱和度) 和"几何图"(高饱和度) |
| **拉普拉斯方差 (Laplacian)** | 测量纹理丰富度，排除纯色背景块 |
| **HSV 色彩分割** | 提取红/蓝/绿色像素，辅助定位几何图中的彩色弧线标记 |
| **比例边距 (15%轮廓尺寸)** | 裁剪时向外扩展 15%，确保字母标签 (A/B/C/D/O) 完整 |

#### 算法演进（8 版迭代）

| 版本 | 方法 | 问题 | 解决 |
|------|------|------|------|
| V1 | 轮廓 + 固定 pad=8px | 缺字母 A/B | → V2 |
| V2 | 膨胀 40px 吞标签 | 仍缺右侧字母 | → V3 |
| V3 | 膨胀 100px | A/B 距离太远 | → V4 |
| V4 | 距离聚类 200px | 吞并了文字段落 | → V5 |
| V5 | 聚类 100px | 漏检几何图本身 | → V6 |
| V6 | 红色弧线定位 | 不可靠（有的图无红色） | → V7 |
| V7 | 饱和度方差 ≥15 | 黑白线段图被过滤 | → V8 |
| **V8** | **饱和度 + 逐字稿辅助** ✅ | **最终稳定版** | — |

#### 最终参数

| 参数 | 值 | 作用 |
|------|-----|------|
| Canny 阈值 | 20-80 | 边缘检测灵敏度 |
| 闭运算核 | 7×7, 2次 | 连接相邻边缘 |
| 轮廓面积 | 0.2%-50% | 过滤过小/过大 |
| 饱度阈值（有"如图"标记的幻灯片）| sat ≥ 5 | 捕获黑白线段图 |
| 饱度阈值（其他幻灯片）| sat ≥ 15 | 严格过滤文字块 |
| 精选面积 | 0.3%-10% | 排除误检 |
| 选图策略 | 最小面积 | 几何图 < 老师 < 整页 |
| 裁剪边距 | 15% 轮廓尺寸 | 包住字母标签 |

> **逐字稿辅助**：通过扫描逐字稿中的"如图"关键词，自动识别哪些幻灯片含有几何图形，对这些幻灯片降低饱和度阈值（从 15 降至 5），从而捕获纯黑白色线条图。

### V3b：讲义生成

| 技术 | 作用 |
|------|------|
| **逐字稿分段** | 按时间三等分（知识导入 / 知识讲解 / 典例剖析），或用"如图"关键词辅助分段 |
| **docx-js** | Node.js 库，代码化生成 `.docx`。支持设置字体（微软雅黑）、纸张（A4）、页眉页脚、代码块灰色背景、图片嵌入 |
| **Pillow (PIL)** | Python 图像库，用于旧版方案中的图片裁剪和压缩 |
